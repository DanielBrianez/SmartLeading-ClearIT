# Contexto — Bootstrap e Análise de Gaps

## Situação Inicial
O projeto ClearIT continha arquivos de documentação clássicos e código funcional Streamlit, mas não possuía a estrutura modular do Onion Portable (`business-context-lite.md`, `technical-context-lite.md`, etc.). Havia incerteza sobre como alinhar o código adiantado do desenvolvedor com os requisitos ideais.

## Motivação
Configurar o ambiente do Onion Portable para habilitar o copiloto lógico na IDE, preservando a documentação e mapeando os desvios de código de forma clara para guiar implementações futuras.

## Restrições
O código atual do ClearIT não deve sofrer alterações ou modificações nesta etapa, respeitando a diretriz do usuário.
